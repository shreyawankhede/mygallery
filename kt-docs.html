Project Overview

The Premium Image Gallery is a high-performance, responsive web application designed with modern standards. It leverages a data-driven approach, allowing for seamless updates and maintenance.

HTML5
CSS3
ES6+ JavaScript
Pro Tip: This gallery is "serverless-ready," meaning it can be hosted on any static hosting service like GitHub Pages, Vercel, or Netlify.

System Architecture

The application follows a modular architecture where content management is decoupled from the UI structure. This ensures that the aesthetic remains consistent regardless of the number of images added.

Folder Structure


Photography/
├── images/             # All gallery images (.jpg, .png)
├── index.html          # Main gallery interface
├── style.css           # Styling system (variables, themes)
├── script.js           # dynamic engine (rendering, modal, logic)
├── kt-docs.html        # Technical documentation (this page)
└── docs-style.css      # Documentation-specific styles

The Gallery Data Engine

At the heart of the application is the galleryData array in script.js. This array acts as a simple database for the images.

script.js

const galleryData = [
    { 
      id: 1, 
      src: 'images/photo.jpg', 
      title: 'Majestic Lion', 
      category: 'wildlife' 
    },
    // ... more items
];

The project dynamically generates the HTML for each image block, ensuring that any changes to the data are reflected instantly across the site without manually editing the HTML for each image entry.

How to Maintain

01 Adding a New Image

1. Upload the image file to the images/ folder.

2. Open script.js and find the galleryData array.

3. Add a new object following the existing format with a unique id.

02 Modifying Categories

If you want to add a new category (e.g., "Architecture"):

1. Add a new <button> in index.html within the filter-bar.

2. Ensure the data-filter attribute on the button matches the category in your galleryData.

Warning: Always ensure image paths (src) are correct and relative to the root folder to avoid broken image links.

Core Features

Smart Filtering

Uses an efficient .filter() method to redraw the gallery grid without page reloads.

Advanced Modal

Handles next/prev navigation using state tracking of the filteredItems array.

Theme Switching

Uses localStorage to remember the user's preference (Dark or Light mode) across sessions.

Typed Interaction

A CSS-inspired typing effect provides a premium "living" feel to the hero section.

Styling System

The system uses 100% Vanilla CSS with CSS Custom Properties (Variables) for easy global theme changes.

style.css

:root {
    --primary-color: #3e82f7;
    --bg-color: #ffffff;
    --text-color: #1a1a1a;
    /* ... more variables */
}

[data-theme="dark"] {
    --bg-color: #0f172a;
    --text-color: #f1f5f9;
}

Responsiveness

The gallery uses a mix of CSS Grid and Flexbox to ensure accessibility on all screens.

  • Desktop: Elegant grid with hover-to-zoom overlays.
  • Tablet: Adjusted grid columns (2-3 per row) for readability.
  • Mobile: Simplified single-column view with touch-friendly navigation.
